# **Проект: Аналитика запуска цифровых продуктов и краудфандинговых кампаний**

## **1\. Описание проекта и бизнес-задача**

Статистика показывает, что значительная часть ИТ-продуктов и краудфандинговых кампаний не достигает целевых показателей из\-за ошибок в позиционировании, неверного выбора ниши или недостаточного взаимодействия с аудиторией на старте.

**Бизнес-задача:** Разработка аналитической стратегии для авторов и компаний, планирующих вывод продукта на рынок.

**Цель исследования:** На основе обработки массивов исторических данных выявить закономерности, определяющие успех запуска. Исследование охватывает текстовые характеристики (длина слоганов), поведенческие факторы (активность комментаторов), влияние кураторов площадок и технические аспекты оформления (видеоконтент, опыт автора).

## **2\. Источники и методы сбора данных**

Исследование базируется на данных из глобальных экосистем для запуска продуктов (Product Hunt) и крупнейшей российской краудфандинговой платформы (Planeta.ru).

### **Методология сбора:**

* **API-интеграция (GraphQL и REST):** Сбор данных о продуктах, пользователях и комментариях осуществлялся через авторизованные запросы к API с использованием библиотеки requests. Применялись сложные GraphQL-запросы для получения глубоко вложенных структур данных (деревья комментариев).  
* **Веб-скрапинг (Web Scraping):** Для получения данных с платформы Planeta.ru использовалась библиотека **Selenium** (для имитации браузера и обхода динамической подгрузки контента) в связке с **BeautifulSoup4**. Это позволило собрать данные, недоступные через стандартные инструменты выгрузки.  
* **Логирование и конфигурация:** Процесс сбора полностью автоматизирован и сопровождается логированием через модуль logging. Настройки параметров (уровни логирования, пути к файлам) вынесены в отдельный конфигурационный файл settings.json, что обеспечивает воспроизводимость и стабильность работы скриптов.

## **3\. Структура данных**

Анализ проводится на базе 6 взаимосвязанных датасетов:

1. **Posts:** Характеристики продуктов, количество голосов и данные о продвижении кураторами (Featured).  
2. **Topics:** Данные о популярности тематических ниш.  
3. **Collections:** Информация о кураторских подборках и их описаниях.  
4. **Users:** Профили авторов, их социальный капитал и внешние ссылки.  
5. **Comments:** База пользовательских отзывов, их содержание и реакции на них.  
6. **Planeta (Crowdfunding):** Финансовые показатели, региональная привязка и специфика оформления проектов.

## **4\. Разведочный анализ данных (EDA)**

В ходе работы выполнено 20 целевых анализов. Перед визуализацией проведена предобработка данных: фильтрация технических ошибок (удаление записей error \== gate), обработка пропусков и удаление дубликатов.

### **Ключевые направления анализа:**

* **Продуктовые метрики:** Корреляционный анализ голосов, отзывов и рейтингов. Оценка влияния "золотого часа" (времени попадания в Featured) на виральность продукта.  
* **Текстовая аналитика:** Распределение слоганов по количеству символов и поиск оптимальной длины "Elevator Pitch" для максимизации внимания.  
* **Рыночные ниши и кураторство:** Анализ концентрации аудитории (закон Парето) в топ-10 темах и коллекциях. NLP-обработка описаний для выявления слов-триггеров в успешных подборках.  
* **Поведенческий анализ:** Исследование зависимости популярности комментария от его длины и выявление часов пиковой активности пользователей для эффективного комьюнити-менеджмента.  
* **Краудфандинговая специфика:** Сравнение сборов в зависимости от категории. Анализ влияния видеопрезентаций и репутации автора (количество созданных ранее проектов) на итоговый успех.

## **5\. Основные бизнес-выводы**

1. **Эффект доверия и опыта:** Авторы, имеющие в портфолио более одного запущенного проекта, показывают медианно более высокие сборы. Это подтверждает важность накопления репутации внутри платформы.  
2. **Влияние медиаконтента:** Наличие видеопрезентации статистически значимо коррелирует с более высоким процентом выполнения финансового плана проекта.  
3. **Роль кураторства:** Попадание в блоки рекомендаций является определяющим фактором успеха. Проекты без поддержки кураторов имеют выраженный "потолок" популярности вне зависимости от качества дискуссии.  
4. **Стратегия взаимодействия:** Ответы автора на комментарии в часы пиковой активности аудитории способствуют росту рейтинга продукта за счет алгоритмического продвижения "живых" обсуждений.
